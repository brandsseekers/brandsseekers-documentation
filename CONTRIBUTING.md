# Contributing to Brands Seekers Documentation

Thank you for your interest in improving the Brands Seekers Documentation.

This repository documents the public machine-readable resources published by Brands Seekers, including product feeds, structured commerce data, multilingual resources, schemas, and integration guides.

## Documentation Principles

All documentation should be:

- Accurate
- Version-aware
- Machine-friendly
- Human-readable
- Technically precise
- Consistent across all specifications

## Documentation Standards

Documentation should:

- Use clear Markdown formatting.
- Explain the purpose before implementation details.
- Include examples where appropriate.
- Avoid undocumented breaking changes.
- Preserve backwards compatibility whenever possible.

## Specifications

When updating specifications:

- Document newly introduced fields.
- Mark deprecated fields clearly.
- Keep examples synchronized with production behavior.
- Update related documentation where applicable.

## Issues

If you discover inaccurate documentation or inconsistencies, please open a GitHub Issue describing the problem and the proposed correction.

## Versioning

Documentation evolves alongside the Brands Seekers platform.

Whenever practical, changes are additive to minimize disruption for existing integrations.
